## 1. 系統架構圖（邊緣 + 雲端 Hybrid）

```mermaid
flowchart TB
    subgraph Edge["感知層 / 邊緣（每處場域一套）"]
        E1["ESP32-S3 #1<br/>ESPectre v2.7.0"]
        E2["ESP32-S3 #2"]
        E3["ESP32-S3 #3"]
        CB["core_bridge.py<br/>(Serial/BLE 接收 + 跌倒偵測)"]
        E1 -->|BLE/Serial| CB
        E2 -->|BLE/Serial| CB
        E3 -->|BLE/Serial| CB
    end

    subgraph Cloud["雲端（Supabase / PostgreSQL）"]
        AUTH["Auth<br/>(密碼雜湊 + Google OAuth)"]
        DB[("關聯式資料庫<br/>residents / fall_events<br/>health / devices ...")]
        RLS["Row-Level Security<br/>角色權限"]
    end

    subgraph Client["表現層（React 前端）"]
        LIVE["即時監控<br/>(10Hz live)"]
        MGMT["管理頁<br/>住民/健康/報表/警報"]
    end

    CB -->|本地 WebSocket 10Hz| LIVE
    CB -->|REST：跌倒事件/活動彙整/心跳| DB
    MGMT <-->|SDK + Realtime| DB
    MGMT --- AUTH
    DB --- RLS
    CB -->|LINE Notify| LINE["LINE 照護人員"]
```

**設計理念**：高頻即時訊號（10Hz movement score）走**本地 WebSocket**確保零延遲；
具管理意義的事件（跌倒、每分鐘活動彙整、裝置心跳）與所有業務資料走**雲端關聯式資料庫**，
支援多裝置、多使用者、跨地點存取。

### 使用案例說明（節錄）

| 編號 | 名稱 | 主要參與者 | 前置條件 | 主要流程 |
|------|------|-----------|---------|---------|
| UC2 | 即時監控 | 醫護/家屬/管理者 | 已登入、裝置在線 | 開啟監控頁 → 訂閱本地 WS → 顯示 10Hz 移動分數、活動狀態、定位 |
| UC3 | 接收跌倒警報 | 醫護/家屬 | 已登入 | 系統偵測跌倒 → 寫入 `fall_events` → 前端即時跳出警報 + LINE 推播 |
| UC4 | 確認/誤報 | 醫護 | 有 pending 警報 | 點「確認」或「誤報」→ 更新事件 status → 供誤報率統計 |
| UC7 | 查看管理報表 | 醫護/家屬/管理者 | 已登入 | 選住民/期間 → 查詢彙整檢視表 → 顯示跌倒統計、活動趨勢、在線率 |
| UC11 | 上傳跌倒事件 | 感測裝置 | 後端持有 service key | 跌倒上升邊緣 → REST insert `fall_events` |

## 5. 循序圖：跌倒偵測 → 警報 → 確認

```mermaid
sequenceDiagram
    participant ESP as ESP32-S3
    participant CB as core_bridge.py
    participant LINE as LINE Notify
    participant DB as Supabase
    participant FE as React 前端
    participant Med as 醫護人員

    ESP->>CB: CSI 移動分數 (10Hz)
    CB->>CB: FallDetector 尖峰判定
    Note over CB: 偵測到跌倒上升邊緣
    par 即時與持久化
        CB->>FE: 本地 WS 廣播 is_falling=true
        CB->>DB: REST insert fall_events (status=pending)
        CB->>LINE: POST 推播（含分數/位置/時間）
    end
    LINE-->>Med: 手機收到跌倒警報
    DB-->>FE: Realtime 推送新事件
    FE-->>Med: 介面跳出紅色警報
    Med->>FE: 點「確認」
    FE->>DB: update fall_events status=confirmed
    Note over DB: 供誤報率與事件統計報表
```

## 6. 權限控管（RLS）對照

| 角色 | residents | fall_events / health | 報表 | 使用者管理 | 偵測設定 |
|------|-----------|----------------------|------|-----------|---------|
| admin | 全部 R/W | 全部 R/W | ✅ | ✅ | ✅ |
| medical | 全部 R/W | 全部 R/W | ✅ | ❌ | ✅ |
| family | 僅綁定住民 R | 僅綁定住民 R | 僅綁定住民 | ❌ | ❌ |
